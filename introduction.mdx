pwndbg (/paʊnˈdiˌbʌɡ/) is a powerful plugin for GDB and LLDB that transforms these debuggers into first-class tools for low-level software analysis. It provides a rich context display, heap inspection commands, memory analysis utilities, and integrations with popular decompilers — everything exploit developers and reverse engineers need without leaving the debugger.

## Why pwndbg?

Vanilla GDB and LLDB are powerful but painful to use for reverse engineering and exploit development. Commands like `x/30gx $rsp` give raw output with no context. pwndbg fills this gap with:

- **Rich context display** — every time execution stops, pwndbg prints registers, disassembly, the stack, and call frames, all color-coded by memory region
- **Heap inspection** — deep visibility into glibc malloc, jemalloc, and musl allocators; visualize chunks, bins, and tcache
- **Memory tools** — telescope pointers recursively, search memory, find leaks, and manipulate ASLR
- **Decompiler integration** — sync with IDA Pro, Binary Ninja, Ghidra, or angr to see decompiled code alongside your debugger
- **Kernel debugging** — inspect SLUB/buddy allocators, BPF programs, kernel tasks, and more via QEMU
- **Cross-debugger** — works with both GDB and LLDB (LLDB support is early-stage)

## Compatibility

| Setup | Supported Versions | Notes |
|---|---|---|
| pwndbg + GDB | Python 3.10+, GDB 12.1+ | Battle-tested on Ubuntu 22.04/24.04 |
| pwndbg + LLDB | Python 3.12+, LLDB 19+ | Experimental/early-stage |
| QEMU user-mode | QEMU 8.1+ | vFile API required for vmmap |
| QEMU system | QEMU 6.2+ | Supported since Ubuntu 22.04 |
